# Linear Issue Creation

## Behavior

Before creating any issue:
1. Analyze the referenced code or description thoroughly
2. Ensure connectivity with Linear MCP
3. Identify and surface any requirement gaps (ambiguous scope, missing edge cases, unclear ownership)
4. Present your gap analysis and the proposed issue draft
5. Always present the synopsis of the changes at each step and ask for confirmation before proceeding to the next step
6. Do not include TypeScript code samples in the issue body.
7. Ask for explicit confirmation before creating or updating the issue on Linear

---

## Gap Analysis (pre-creation)

Before drafting, check for and flag:
- [ ] Unclear or missing acceptance criteria
- [ ] Ambiguous scope boundaries (what's in vs. out)
- [ ] Unidentified dependencies or affected systems
- [ ] Missing non-functional requirements (performance, security, accessibility)
- [ ] No clear owner or team assignment

Present gaps as questions, e.g.:
> "Before I create this issue, I need to clarify: How does the behavior differ for each user role?
> What are the impacted components within the codebase?
> Are there any performance requirements we should consider?
> Any other queries?"
